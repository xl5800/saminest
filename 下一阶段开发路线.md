# DMV 华人集市下一阶段开发路线

## 当前状态

平台已经完成 MVP 核心链路：

- 用户登录 / 注册 / 忘记密码
- 租房和二手发布
- Supabase 数据库存储
- Supabase Storage 图片上传
- 管理员审核
- 审核通过后前台展示
- 我的发布
- 多图上传基础版本
- DMV 地区筛选和自定义地区

现在的问题是：功能已经跑通，但代码和产品形态还比较原型化。下一阶段目标不是继续堆功能，而是把平台整理成可维护产品。

## 第一阶段：稳定发布体验

目标：让用户发布租房和二手时感觉专业、清楚、可信。

要做：

1. 发布表单继续按类型拆分
   - 租房只显示租房字段
   - 二手只显示二手字段
   - 必填项要明确
   - 手机端表单不要太长

2. 多图上传完善
   - 最多 6 张
   - 第一张封面
   - 图片预览
   - 删除单张图
   - 上传失败提示清楚

3. 详情页完善
   - 多图展示
   - 联系方式复制
   - 风险提醒
   - 举报按钮
   - 发布时间和更新时间

## 第二阶段：整理数据库

目标：避免所有信息都塞进 `listings.description`，后面不好筛选。

建议表结构：

```text
listings
listing_images
rental_details
secondhand_details
reports
favorites
profiles
```

下一步优先加：

```text
rental_details
secondhand_details
```

这样租房字段和二手字段可以独立管理。

## 第三阶段：拆分前端代码

当前 `app.js` 已经超过 1000 行，不适合继续堆功能。

建议拆成：

```text
js/config.js
js/state.js
js/auth.js
js/listings.js
js/upload.js
js/forms.js
js/admin.js
js/render.js
js/maps.js
js/main.js
```

拆分原则：

- 每次只拆一个模块
- 拆完后立刻测试登录、发布、审核、展示
- 不在重构时顺手加新功能

第一步建议先拆：

```text
auth.js
upload.js
forms.js
```

因为这三个模块边界最清楚。

## 第四阶段：后台审核升级

目标：管理员不只是点通过/拒绝，而是能判断风险。

要做：

- 查看详情
- 查看图片
- 修改分类
- 标记可疑
- 下架
- 查看举报
- 举报处理状态

## 第五阶段：用户信任系统

目标：减少诈骗和低质量内容。

要做：

- 邮箱已验证
- 手机号验证，后续加
- 举报原因
- 可疑关键词提醒
- 联系方式登录后可见
- 发布频率限制

## 第六阶段：上线运营准备

上线前检查：

- Supabase RLS 是否开启
- 只有 approved 帖子公开
- 图片 bucket 权限是否正确
- Vercel 最新部署是否成功
- 手机端是否可用
- 发布流程是否少于 3 分钟
- 管理员账号是否正常

## 推荐下一次开发顺序

1. 先做数据库 `rental_details` 和 `secondhand_details`
2. 再把发布表单的数据存到对应详情表
3. 然后拆 `app.js` 的 `auth.js`
4. 再拆 `upload.js`
5. 最后优化手机端 UI

每次只做一件事，做完测试，再进入下一步。

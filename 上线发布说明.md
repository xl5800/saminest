# Saminest 上线发布说明

## 线上组件

- Supabase：登录、数据库、图片存储、审核、举报和站内消息
- Vercel 或其他静态托管：部署网站文件
- 域名：`https://www.saminest.com`

## Supabase 初始化

1. 在 Supabase 创建项目。
2. 在 SQL Editor 依次运行：
   - `supabase-schema.sql`
   - `supabase-messaging-and-policies.sql`
   - `supabase-production-updates.sql`
3. 用你的邮箱注册一次账号。
4. 在 SQL Editor 把你的账号设为管理员：

```sql
update public.profiles
set role = 'admin'
where email = '你的邮箱@example.com';
```

## 前端配置

把 `supabase-config.example.js` 复制成 `supabase-config.js`，只填写 Supabase Project URL 和 anon public key：

```js
window.SAMINEST_SUPABASE_CONFIG = {
  url: "你的 Supabase Project URL",
  anonKey: "你的 Supabase anon public key",
};
```

注意：前端只能放 anon public key。不要把 service_role key、SMTP 密码、Resend API Key 放到任何前端文件里。

## Supabase Auth 设置

在 Supabase Dashboard 设置：

- Authentication > Providers > Email：开启 Confirm email。
- Authentication > URL Configuration > Site URL：`https://www.saminest.com`
- Authentication > URL Configuration > Redirect URLs：
  - `https://www.saminest.com/**`
  - `https://saminest.com/**`
- Authentication > SMTP Settings：发件邮箱和 SMTP 账号在这里配置，例如 `noreply@saminest.com`。

发件邮箱由 Supabase SMTP 控制，前端不控制发件邮箱。

## 上线前重点测试

- 注册：收到确认邮件，点击后能回到网站登录。
- 登录 / 退出：登录状态和底部导航显示正常。
- 忘记密码：邮件链接直接进入设置新密码页面，设置成功后退出到登录状态。
- 发布：租房、求租、二手发布后普通用户进入待审核，管理员可审核通过。
- 图片：图片上传到 `listing-images` bucket，并保存到 `listings.image_url` 和 `listing_images`。
- 我的发布：用户只能编辑、下架、删除自己的帖子。
- 举报：用户可提交举报，管理员可在举报管理查看。
- 移动端：首页、分类、发布、消息、我的页面底部导航不遮挡内容。

## 安全检查

- RLS 保持开启。
- 普通用户只能创建自己的 pending 帖子。
- 普通用户编辑已通过帖子后会重新进入 pending，避免绕过审核。
- 用户只能管理自己的收藏、消息和发布内容。
- 管理员权限只通过 `profiles.role = 'admin'` 控制。
